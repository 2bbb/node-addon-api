1 # Node.js API (N-API) Package
2 
3 This package contains header-only C++ wrapper classes for the ABI-stable
4 Node.js API (N-API), along with library code that enables
5 backward-compatibility with use with older versions of Node.js that do
6 not have N-API built-in.
7 
8 To use N-API in a native module:
9  1. Add a dependency on this package to `package.json`.
10  It is not yet published to npm, so reference it directly from GitHub.
11 ```json
12  "dependencies": {
13  "node-api": "github:nodejs/node-api",
14  }
15 ```
16 
17  2. Reference this package's include directory and gyp file in `binding.gyp`:
18 ```gyp
19  'include_dirs': ["<!(node -p \"require('node-api').include\")"],
20  'dependencies': ["<!(node -p \"require('node-api').gyp\")"],
21 ```
22 
23  3. Ensure C++ exceptions are enabled in `binding.gyp`.
24  The N-API C++ wrapper classes use exceptions for error-handling;
25  the base ABI-stable C APIs do not.
26 ```gyp
27  'cflags!': [ '-fno-exceptions' ],
28  'cflags_cc!': [ '-fno-exceptions' ],
29  'xcode_settings': { 'GCC_ENABLE_CPP_EXCEPTIONS': 'YES' },
30  'msvs_settings': {
31  'VCCLCompilerTool': { 'ExceptionHandling': 1 },
32  },
33 ```
34 
35  4. Include `napi.h` in the native module code.
36  To ensure only ABI-stable APIs are used, DO NOT include `node.h` or `v8.h`.
37 ```C++
38 #include "napi.h"
39 ```
40 
41 At build time, the N-API back-compat library code will be used only when the targeted node version *does not* have N-API built-in.
42

44 for more details about N-API.
